    KMP 算法

    简介$

    \(KMP\) 算法并不能用来看毛片，它是一种字符串匹配算法。\(KMP\) 是该算法的发明者 \(Kruth\)，\(Morris\) 和 \(Pratt\) 的名字缩写。

    引出问题$

    给定字符串 \(A\)（长度为 \(m\)）和 \(B\)（长度为 \(n\)），问 \(A\) 中是否包含 \(B\) ？

    首先将 \(A\) 和 \(B\) 的左端对齐，发现 \(\textcolor{red}{a}\) 与 \(\textcolor{red}{c}\) 不匹配，于是将 \(B\) 向右移一位。

    重复右移 \(B\)，直到在 \(A\) 中匹配出完整的 \(B\)。

    暴力算法
    bool involve(string a, string b) {
    int m = a.size(), n = b.size();
    int i = 0, j = 0;
    while(i < m) {
        while(j < n && a[i] == b[j]) j ++;
        if(j == n) return true;
        i ++, j = 0;
    }
    return false;
}
    暴力算法（\(\textcolor{red}{×}\)） \(KMP\) 算法（\(\textcolor{green}{√}\)）
    \(\textcolor{red}{O(mn)}\) \(\textcolor{green}{O(m+n)}\)

    基本原理$

    其实，刚才匹配到这一步时，\(B\) 串可以直接向后移 \(2\) 位，从黄色处开始比较。

    为什么可以这么做呢？

    首先，观察得出 \(B\) 串的这两段相等。

    而此时 \(A\) 串和 \(B\) 串的绿色部分相等，框出的这两段也相等。

    因此 \(A\) 串和 \(B\) 串的这两段也相等。

    容易证明，若 \(B\) 串只向右移 \(1\) 位，肯定匹配不成。因此 \(B\) 串至少需要向右移 \(2\) 位。

    下面一例中，容易证明 \(B\) 串在右移 \(1\sim 3\) 位时都匹配不成，因此至少向右移 \(4\) 位。

    即当不匹配时，从 \(B\) 串的绿色子段中，找出前后两段相等的，再把 \(B\) 串右移，直到到这两段位于同一竖直位置，再从黄色处继续匹配（配合下图理解）。

    \(KMP\) 算法快就快在这。

    预处理$

    当务之急是，在不匹配时，从 \(B\) 串的绿色子段中找出相同的「前缀」和「后缀」。

    下面一例中，\(\textcolor{green}{joxjo}\) 的前缀（\(\textcolor{green}{jo}\)）和后缀（\(\textcolor{green}{jo}\)）相同。\(\textcolor{green}{jo}\) 称为 \(\textcolor{green}{joxjo}\) 的「公共前后缀」。

    注意：公共前后缀不能和原字符串相等。即 \(joxjo\) 的公共前后缀不能是 \(joxjo\)。

    \(next[i]\)：\(B[0\cdots i]\) 的最长公共前后缀的长度。

    \(i\) \(B[0\cdots i]\) 公共前后缀 \(next[i]\)
    \(0\) \(j\) \(0\)
    \(1\) \(jo\) \(0\)
    \(2\) \(jox\) \(0\)
    \(3\) \(\underline{\textcolor{red}{j}}ox\underline{\textcolor{red}{j}}\) \(\textcolor{red}{j}\) \(1\)
    \(4\) \(\underline{\textcolor{red}{jo}}x\underline{\textcolor{red}{jo}}\) \(\textcolor{red}{jo}\) \(2\)
    \(5\) \(\underline{\textcolor{red}{jox}}\underline{\textcolor{red}{jox}}\) \(\textcolor{red}{jox}\) \(3\)
    \(6\) \(joxjoxx\) \(0\)
    \(7\) \(joxjoxxx\) \(0\)

    如何编程求出 \(next[ \ ]\) 数组？这里以字符串 \(aabaabaaa\) 为例。

    对于任意字符串 \(B\)，\(next[0]=0\)。

    \(x\) \(0\) \(1\) \(2\) \(3\) \(4\) \(5\) \(6\) \(7\) \(8\)
    \(next[x]\) \(\textcolor{red}{0}\)

    使 \(i\) 指向 \(B[0]\)，\(j\) 指向 \(B[1]\)，即 \(i=0,j=1\)。

    由于 \(B[i]=B[j]\)，因此 \(next[1]=1\)。

    \(x\) \(0\) \(1\) \(2\) \(3\) \(4\) \(5\) \(6\) \(7\) \(8\)
    \(next[x]\) \(0\) \(\textcolor{red}{1}\)

    \(i\) 和 \(j\) 同时向后移一位，看下一位是否也相等。

    \(B[i]≠B[j]\)，出现了不匹配的情况，\(next[2]=0\)。

    \(x\) \(0\) \(1\) \(2\) \(3\) \(4\) \(5\) \(6\) \(7\) \(8\)
    \(next[x]\) \(0\) \(1\) \(\textcolor{red}{0}\)

    \(i\) 移回 \(0\)，\(j\) 再向后移一位。

    \(B[i]=B[j]\)，\(next[3]=1\)。

    \(x\) \(0\) \(1\) \(2\) \(3\) \(4\) \(5\) \(6\) \(7\) \(8\)
    \(next[x]\) \(0\) \(1\) \(0\) \(\textcolor{red}{1}\)

    \(i\) 和 \(j\) 同时向后移一位。

    \(B[i]=B[j]\)，\(next[4]=next[3]+1=2\)。

    \(x\) \(0\) \(1\) \(2\) \(3\) \(4\) \(5\) \(6\) \(7\) \(8\)
    \(next[x]\) \(0\) \(1\) \(0\) \(1\) \(\textcolor{red}{2}\)

    \(i\) 和 \(j\) 同时向后移一位。

    \(B[i]=B[j]\)，\(next[5]=next[4]+1=3\)。

    \(x\) \(0\) \(1\) \(2\) \(3\) \(4\) \(5\) \(6\) \(7\) \(8\)
    \(next[x]\) \(0\) \(1\) \(0\) \(1\) \(2\) \(\textcolor{red}{3}\)

    \(i\) 和 \(j\) 同时向后移一位。

    \(\cdots \ \cdots\)

    就这样一直比较到最后一位，再次出现了不匹配的情况。所以 \(next[8]\) 应该填 \(0\) 吗？显然不是。

    查询表格得到 \(next[7]=5\)，这对应图中的两个绿色子段相等。

    查询表格得到 \(next[i-1]=next[4]=2\)，即图上的这两段相等。

    这代表 \(B\) 串中的这四段都相等。

    于是令 \(i=next[i-1]=2\)，如果 \(B[i]=B[j]\)，则 \(next[8]=i+1=3\)。但是它们并不相等。

    于是我们完全可以故技重施，再次令 \(i=next[i-1]=1\)，结果发现这一次 \(B[i]=B[j]\)。

    因此 \(next[8]=i+1=2\)。

    \(x\) \(0\) \(1\) \(2\) \(3\) \(4\) \(5\) \(6\) \(7\) \(8\)
    \(next[x]\) \(0\) \(1\) \(0\) \(1\) \(2\) \(3\) \(4\) \(5\) \(\textcolor{red}{2}\)

    模板$

    代码
    int next[];

void getnext(string str) {
    next[0] = -1;
    int i = 1, j = 0;
    while(i < str.size()) {
        while(j > -1 && str[i] != str[j]) j = next[j];
        next[++ i] = ++ j;
    }
}

void kmp(string str, string substr) {
    getnext(substr);
    int i = 0, j = 0;
    while(i < str.size()) {
        while(j > -1 && str[i] != substr[j]) j = next[j];
        if(j == substr.size() - 1)
            printf("Fount pattern at %d\n",i - j), j = next[j] - 1;
        i ++, j ++;
    }
}
