    如何利用位运算薅羊毛

    May 15, 2021
    算法   位运算  

    简介 $

    $C++$ 提供了 $6$ 个快到极致的位运算操作符（不会的请自行百度）：

    操作符 $\&$ $|$ $∧$ $\sim$ $«$ $»$
    读作 与 或 异或 取反 左移 右移

    位运算有多块？在 $32$ 位的 $CPU$ 中，位运算比普通的乘除法快约 $40$ 倍。位运算凭此成为 $OI$ 赛场上的薅羊毛利器。

    重要性质 $

    • $a\; \& \; 0=0$

    • $a\; \& -1=a$

    • $a\; | \; 0=a$

    • $a\; | -1=-1$

    • $a∧a=0$

    • $a∧0=a$

    • $a\; «\; 1=a×2$

    • $a\; »\; 1=a÷2$（向下取整）

    注：$-1$ 在内存中表示为 $11111111$（每一位都为 $1$）。

    小数转整数 $

    正常写法：

    x = int(x);

    位运算优化：

    x = x >> 0;

    相反数转换 $

    正常写法：

    x = -x;

    位运算优化：

    x = ~x + 1;

    判断奇偶性 $

    判断 $x$ 是否为奇数，只需要看 $x÷2$ 的余数。

    $$ \text{isOdd}(x)=\left\{\begin{aligned} &\text{true}, && x \ \text{mod} \ 2=1\\ &\text{false}, && x \ \text{mod} \ 2=0 \end{aligned}\right. $$

    将 $x$ 和 $1$ 进行「与」运算，可以取出 $x$ 在二进制下的第一位。举个例子：

    $$ \begin{aligned} 1 \ 0 \ 0 \ 1 \ 0 \ 1 \ 1 \ 1& \\ \text{\&} \ 0 \ 0 \ 0 \ 0 \ 0 \ 0 \ 0 \ 1& \\ \overline{ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ }&\\ 0 \ 0 \ 0 \ 0 \ 0 \ 0 \ 0 \ 1& \end{aligned} $$

    取出的这一位数字正是 $x÷2$ 的余数。于是：

    $$ \text{isOdd}(x)=\left\{\begin{aligned} &\text{true}, && x \; \& \; 1=1\\ &\text{false}, && x \; \& \; 1=0 \end{aligned}\right. $$
    bool isOdd(int x) { // x 为奇数则返回 true，否则 false
    return x & 1;
}

    判断是否同号 $

    在内存中，正数的最高位为 $0$，负数的最高位为 $1$。

    $$ \begin{aligned} (int)&+14→0000\cdots 1110\\ (int)&-14→\underbrace{1111\cdots 0010}_{共 \ 32 \ 位} \end{aligned} $$

    而 $int$ 类型的变量一共有 $32$ 位。于是将变量右移 $31$ 位，便可提取出它的最高位。

    $$ \begin{aligned} &+14\;»\;31=0000\cdots 1110\;»\;31=0\\ &-14\;»\;31=\underbrace{1111\cdots 0010}_{共 \ 32 \ 位}\;»\;31=1 \end{aligned} $$
    bool judge(int a, int b) {
    return (a >> 31) == (b >> 31);
}

    还可以写得更高逼格一点：

    bool judge(int a, int b) {
    return !((a ^ b) >> 31);
}

    快速读入 $

    原理简洁易懂：

    $$x×10=x×2+x×8=(x\;«\;1)+(x\;«\;3)$$

    int read() { // 读入一个整数
    int x = 0, f = 1; char s = getchar();
    while(s < '0' || s > '9') {
        if(s == '-') f = -f;
        s = getchar();
    } while(s >= '0' && s <= '9') {
        x = (x << 1) + (x << 3) + s - '0'; // 等价于 x = x * 10 + s - '0'
        s = getchar();
    } return x * f;
}

    求绝对值 $ 

    int abs(int x) {
    return (x ^ (x >> 31)) - (x >> 31);
}

    求最大/最小值 $

    通常我们会这样定义 $min(a, b)$：

    int min(int a, int b) { // a 和 b 谁最小就返回谁
    if(a < b) return a; // if 语句
    else return b;
}

int min(int a, int b) {
    return a < b ? a : b; // 三目运算符
}

    但在某些机器上，执行「$if()$ 语句」或「三目运算符」很浪费时间，于是有人想出了更快的版本：

    int min(int a, int b) {
    return b ^ ((a ^ b) & - (a < b));
}

    乍一看还挺神奇。现在我们逐步理解该语句。首先看最右边的 $-(a<b)$：

    $$ -(a < b) =\left\{\begin{aligned} &-1, &&a < b\\ &0, &&a≥b \end{aligned}\right. $$

    根据 重要性质，有

    $$ (a∧b)\&\color{blue}{-(a < b)}\color{black} =\left\{\begin{aligned} &(a∧b)\;\&\;\color{blue}{-1}\color{black} =a∧b, &&a < b\\ &(a∧b)\;\&\;\color{blue}{0}\color{black} =0, &&a≥b \end{aligned}\right. $$

    因此

    $$ b∧\color{blue}{((a∧b)\&-(a < b))}\color{black}=\left\{\begin{aligned} & b∧\color{blue}{(a∧b)}\color{black} =a, &&a < b\\ & b∧\color{blue}{0}\color{black} =b, &&a≥b \end{aligned}\right. $$

    因此

    $$min(a, b)=b∧((a∧b)\&-(a<b))$$

    同理，可以推出 $max(a, b)$：

    int max(int a, int b) {
    return a ^ ((a ^ b) & - (a < b));
}

    交换两个变量的值 $

    交换 $a,b$ 的值，一般需要用到「第三个变量」：

    void swap(int& a, int& b) {
    int c; // 第三个变量
    c = a, a = b, b = c;
}

    利用二进制原理，可以省去「第三个变量」占用的内存：

    void swap(int& a, int& b) {
    a ^= b ^= a ^= b;
    // 这条语句等价于 a ^= b, b ^= a, a ^= b;
}

    举个例子就能秒懂：

    $a$ $b$
    初始值 $1$ $2$
    $a\;∧\!=b$ $1∧2$ $2$
    $b\;∧\!=a$ $1∧2$ $2∧1∧2$
    $a\;∧\!=b$ $1∧2∧2∧1∧2$ $2∧1∧2$
    计算结果 $2$ $1$

    判断是否为 2 的幂 $

    判断整数是否是 $2$ 的幂，大部分人会这么写：

    bool judge(int x) {
    int a = 1;
    while(a <= x) {
        if(a == x) return true;
        a *= 2;
    }
    return false;
}

    时间复杂度为 $O(\log{n})$。但是还有人嫌它太慢。

    于是互联网上的高手给出了数个 $O(1)$ 的解法：

    bool judge(int x) {
    return x && !(x & (x - 1));
}

bool judge(int x) {
    return !((1 << 31) % x);
}

bool judge(int x) { // 用到了 lowbit() 的知识
    return x == (x & -x);
}

...

    都是很浅显易懂的写法，然而没有多少人能在第一时间想到 $\cdots \ \cdots$

    向上舍入到 2 的幂 $

    定义 $\text{nextPow}(x)$ 为 $x$ 向上舍入到 $2$ 的幂：

    $$ \begin{aligned} &\text{nextPow}(1)=2\\ &\text{nextPow}(2)=2\\ &\text{nextPow}(3)=4\\ &\text{nextPow}(4)=4\\ &\text{nextPow}(5)=8\\ &\text{nextPow}(6)=8\\ &\text{nextPow}(7)=8\\ &\qquad \cdots \ \cdots \end{aligned} $$

    $O(\log{n})$ 做法：

    int nextPow(int x) {
    int a = 1;
    while(a < x) {
        a *= 2;
    }
    return a;
}

    $O(1)$ 做法：

    int nextPow(int x) {
    x --;
    x |= x >> 1;
    x |= x >> 2;
    x |= x >> 4;
    x |= x >> 8;
    x |= x >> 16;
    x ++;
    return x;
}

    首先看中间最魔幻的几步：

    • $x\;|=x\;»\;1$：使 $x$ 的最高 $2$ 位都为 $1$；

    • $x\;|=x\;»\;2$：使 $x$ 的最高 $4$ 位都为 $1$；

    • $x\;|=x\;»\;4$：使 $x$ 的最高 $8$ 位都为 $1$；

      $\cdots \ \cdots$

    • ${10000001}\;|\;{01000000}=\color{red}{11}\color{black}000001$；

    • ${11000001}\;|\;{00110000}=\color{red}{1111}\color{black}0001$；

    • ${11110001}\;|\;{00001111}=\color{red}{11111111}$；

      $\cdots \ \cdots$

    一顿操作下来，$x$ 的每一位全部都化为 $1$。此时 $x+1$ 便为 $\text{nextPow(}x)$。

    这么做还存在一个问题：如果 $x$ 本来就是 $2$ 的幂，那么 $\text{nextPow(}x)$ 应该等于 $x$，但按照这种做法将得到 $x×2$。于是我们需要在最开始令 $x$ 自减 $1$，即 x --。
