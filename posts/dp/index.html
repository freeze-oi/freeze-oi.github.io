    简介

    基本原理$

    动态规划（DP）是求解多阶段问题的一种策略，求解过程分为以下 5 步。

    1.判断问题和子问题$

    • 把一个主问题分成多个子问题，先求解子问题，再通过子问题求解主问题。

      例题斐波那契数列的主问题是求 \(f[i]\)，子问题是求 \(f[i-1]\) 和 \(f[i-2]\)。

    2.确定状态转移方程$

    • 状态转移方程用于描述问题和子问题之间的关系。

      例题的状态转移方程是 \(f[i]=f[i-1]+f[i-2]\)。

    3.判断初始/边界条件$

    • 初始条件是状态转移方程的预备条件，先于状态转移而存在。

      \(f[1]=f[2]=1\) 是初始条件，状态转移从 \(f[3]\) 开始（\(f[3]=f[1]+f[2]\)）。

    • 边界条件是状态转移的结束条件。

      若只要求到斐波那契数列的第 \(n\) 项，则 \(i=n\) 是边界条件。\(i\) 超过 \(n\) 时应结束状态转移。

    4.判断计算顺序$

    • 合理的计算顺序必须使每个问题的子问题都先于它被求解。

    • 若难以判断计算顺序，可以使用记忆化搜索。

      显然，对于例题，需要从 \(f[3]\) 顺次计算到 \(f[n]\)。

    5.代码实现$

    Code
    f[1] = f[2] = 1;
for(int i = 3; i <= n; i ++)
    f[i] = f[i - 1] + f[i - 2];

    重要性质$

    最优子结构性$

    • 用 DP 处理最优化问题时，一个问题的最优解只取决于其子问题的最优解，与非最优解无关。

    无后效性$

    • 状态 \(A\) 的解一旦被确定，则之后的状态不再被状态 \(A\) 之前的状态所影响。

      \(f[i-1]\) 和 \(f[i-2]\) 的值一旦被确定，则 \(f[i]\) 的值只与它们两个有关，与 \(f[1\cdots i-3]\) 无关。

    子问题重叠性$

    • 多个问题有可能包含同一个子问题。

      \(f[i]=f[i-1]+f[i-2]\)，\(f[i-1]=f[i-2]+f[i-3]\)，

      因此 \(\lceil\)求 \(f[i]\rfloor\) 和 \(\lceil\)求 \(f[i-1]\rfloor\) 这两个问题都包含 \(\lceil\)求 \(f[i-2]\rfloor\) 这一子问题。
