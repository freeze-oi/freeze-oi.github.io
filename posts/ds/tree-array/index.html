    树状数组

    简介$

    树状数组巧妙地利用树结构，优化了单点修改和区间查询的效率。

    • 支持修改数组单个元素的值；

    • 支持快速查询数组某区间的元素和（或积）。

    前置知识：\(\rm lowbit\) 运算。

    基本原理$

    引入问题$

    Question

    题目描述

    给出一个长度为 \(n\) 的数组 \(A\)，进行以下两种操作若干次：

    1. 将第 \(x\) 个数加上 \(k\)；

    2. 输出区间 \([l,r]\) 内每个数的和，即 \(A\) 的第 \(l\sim r\) 个元素的和。

    输入格式

    第一行两个整数 \(n\)（数组元素个数）和 \(m\)（操作次数）；

    第二行 \(n\) 个整数，第 \(i\) 个数表示 \(A_i\)；

    接下来 \(m\) 行每行 \(3\) 个整数，表示一个操作，具体如下：

    • 1 x k：将 \(A_x\) 加上 \(k\)；

    • 2 l r：输出区间 \([l,r]\) 内每个数的和（区间查询）。

    输出格式

    换行输出所有操作 \(2\) 的结果。

    输入样例

    5 5
1 5 4 2 3
1 1 3
2 2 5
1 3 -1
1 4 2
2 1 4

    输出样例

    14
16

    数据范围

    \(1≤n,m≤5×10^5\)。

    暴力算法：

    • 单点修改：直接更改原数组中 \(A_x\) 的值；

    • 区间查询：遍历整个区间 \([l,r]\) 内的元素，累加。

    Code
    int main() {
    cin >> n >> m;
    for(int i = 1; i <= n; i ++) cin >> a[i];
    while(m --) {
        cin >> op;
        if(op == 1) {
            cin >> x >> k; a[x] += k; // 单点修改
        } else {
            int sum = 0;
            cin >> l >> r;
            for(; l <= r; l ++) // 遍历区间 [l, r] 的元素
                sum += a[l];
            cout << sum << endl;
        }
    }
}

    时间复杂度：

    • 单点修改：\(O(1)\)；

    • 区间查询：\(O(n)\)。

    \(m\) 次操作的总复杂度为 \(O(nm)\)。该做法显然接受不了大数据。我们需要使用「树状数组」来苟过所有的测试点。

    树状数组的时间复杂度：

    • 单点修改：\(O(\log{n})\)；

    • 区间查询：\(O(\log{n})\)。

    \(m\) 次操作的总复杂度为 \(O(m\log{n})\)，可以跑非常大规模的数据。

    构造$

    在数组 \(A\) 的上方建立起这样的森林结构：

    其中 \(C_i\) 的值为「以 \(C_i\) 为根节点的子树中，叶结点值的和」，即：

    \(C_1=A_1\)；

    \(C_2=A_1+A_2\)；

    \(C_3=A_3\)；

    \(C_4=A_1+A_2+A_3+A_4\)；

    \(C_5=A_5\)；

    \(C_6=A_5+A_6\)；

    \(C_7=A_7\)；

    \(C_8=A_1+A_2+A_3+A_4+A_5+A_6+A_7+A_8\)。

    \(\cdots \ \cdots\)

    每个节点覆盖的长度为多少呢？将 \(C\) 数组所有元素的下标转换成二进制数，再分别计算它们的 \(\rm lowbit\) 值：

    会发现同一层所有节点的下标 \(\rm lowbit\) 值相同，且节点 \(C_i\) 覆盖的长度恰好为 \(\rm lowbit(i)\)。

    通过进一步观察发现：\(C_i\) 的父节点为 \(C[i+\rm lowbit(i)]\)。例如：

    \(C[\)0010\(]\) 的父节点 \(=C[\)0010 \(+ \rm \ lowbit(\)0010\()]=C[\)0010 \(+\) 10\(]=C[\)0100\(]\)

    单点修改$

    将 \(A_x\) 的值增加 \(k\)，哪些节点的值会跟着变动？以 \(A_3\) 为例：

    \(C_3=\textcolor{red}{A_3}\)；

    \(C_4=A_1+A_2+\textcolor{red}{A_3}+A_4\)；

    \(C_8=A_1+A_2+\textcolor{red}{A_3}+A_4+A_5+A_6+A_7+A_8\)。

    因此需要将 \(C_3,C_4,C_8\) 的值都加上 \(k\)，而 \(C_3,C_4,C_8\) 满足「后一个是前一个的父节点」。

    对于任意的 \(x\)，从 \(C_x\) 开始一层层遍历父节点，并将其值加上 \(k\)。时间复杂度为 \(O(\log{n})\)。

    Code
    int lowbit(int x) {
    return x & -x;
}

void add(int p, int k) { // 将 a[p] 增加 k，同时维护树的构造
    a[p] += k;
    for(; p; p += lowbit(p)) c[p] += k;
}

    前缀查询$

    树状数组支持查询前缀和，即数组 \(A\) 第 \(1\sim x\) 个元素的和。

    \(sum[x]\) 表示 \(A[1\cdots x]\) 所有元素的和。以 \(sum[7]\) 为例：

    \(C_4=A_1+A_2+A_3+A_4\)；

    \(C_6=A_5+A_6\)；

    \(C_7=A_7\)；

    因而 \(C_4+C_6+C_7=sum[7]\)。而 \(C_7,C_6,C_4\) 满足「后一个是前一个的左上方最近节点」。

    同样采用二进制思想，将节点编号转换为二进制数：

    进一步观察发现，\(C_i\) 的左上方最近节点为 \(C[i-\rm lowbit(i)]\)。例如：

    \(C[\)0111 \(- \rm \ lowbit(\)0111\()]=C[\)0111 \(-\) 01\(]=C[\)0110\(]\)

    \(C[\)0110 \(- \rm \ lowbit(\)0110\()]=C[\)0110 \(-\) 10\(]=C[\)0100\(]\)

    查询 \(sum[x]\) 时，将 \(sum[x]\) 的初始值设为 \(0\)，从 \(C_x\) 开始依次遍历左上方最近节点，并将其值累加在 \(sum[x]\) 上。时间复杂度为 \(O(\log{n})\)。

    Code
    int ask(int p) { // 查询 A[1...p] 的和
    int sum = 0;
    for(; p; p -= lowbit(p)) sum += c[p];
    return sum;
}

    区间查询$

    采用前缀和思想。\(get[l,r]\) 表示 \(A[l\cdots r]\) 所有元素的和，则：

    \[ get[l,r]=sum[r]-sum[l-1]; \]
    Code
    int get(int l, int r) { // 查询 A[l...r] 的和
    return ask(r) - ask(l - 1);
}
